# CS300---DSA-Analysis-and-Design


**Overview**

The project focused on designing and implementing the ABCU Advising Program, an application that stores, searches, and displays Computer Science course information. The program loads course data from a CSV file, organizes the data using a Binary Search Tree (BST), prints courses in alphanumeric order using an in-order traversal, and displays prerequisite information for individual courses.


**What was the problem you were solving in the projects for this course?**

For the Data Structures and Algorithms projects, I worked on organizing, sorting, and retrieving computer science course information from a course catalog. In Project One, I developed pseudocode and analyzed the Big O runtime for different data structures. In Project Two, I used that plan to create a working console application that could read course information from a CSV file, parse the data, store it in a structure such as a Binary Search Tree, and allow users to view all courses or search for a specific course and its prerequisites.

**How did you approach the problem? Consider why data structures are important to understand.**

I approached the project by breaking it into smaller steps. I compared the three data structures, vectors, hash tables, and binary search trees, to determine which would best support storing, searching, and sorting the course information. Then I focused on completing each task: reading the file, organizing the course data, displaying the full list in alphanumeric order, and retrieving the title and prerequisites for a selected course. This process helped me understand why data structures are so important. The way data is stored affects how efficiently a program can access, sort, and manage that information.

**How did you overcome any roadblocks you encountered while going through the activities or project?**

One of the main challenges I faced was troubleshooting errors when the code did not behave as expected. I overcame these roadblocks by reviewing the problem step by step, testing each sections of code, reading error messages, and researching possible solutions when needed. Working through these issues helped me become more confident in debugging and problem solving.

**How has your work on this project expanded your approach to designing software and developing programs?**

I understanding why some data structures are better than others depending on the project.  I have not worked with data structures before.  Now I'll consider runtime complexity, scalability, maintainability, and overall program design during the planning stage.  

**How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?**

This experience will help me write programs that are more maintainable and adaptable by organizing code into smaller, manageable sections and a clear structure. I repeatedly referenced past assignments involving code to help with this project.  I was able to reuse code that had similar logic and adapt it for project 2.  I will work on keeping my programs organized, easy to read, and easy to update. This experience showed me how important planning, designing, and testing are when building software.
